**📅 日期**: 2026-03-14上午
**📁 文件**: [[05-网络安全课程/① 上午 - 网络基础与设备.md]]

#### 知识点

| 主题 | 说明 |
|-----|------|
| **网络重要性** | 网络是安全的基础载体，没有网络就没有网络安全 |
| **网卡** | 有线网卡（电信号）vs 无线网卡（电磁波） |
| **光猫** | 光信号/电信号转换，不具备路由功能 |
| **路由器** | 家庭上网必配，比光猫信号强 |
| **交换机** | 企业网络必备，连接多台设备 |
| **核心交换机** | 大公司用，影响范围大 |
| **Wi-Fi** | 2.4G穿墙强但慢，5G速度快但穿墙弱 |

#### 重点概念

- **网络重要性**："没有网络就没有网络安全"
- **网卡功能**：电信号/电磁波信号转换
- **光猫**：光信号/电信号转换，不具备路由功能
- **路由器**：家庭上网必配，比光猫信号强
- **交换机**：企业网络必备，连接多台设备

### ② - 子网掩码与DHCP

**📅 日期**: 2026-03-21
**📁 文件**: [[05-网络安全课程/② - 子网掩码与DHCP.md]]

#### 知识点

| 主题 | 说明 |
|-----|------|
| **子网掩码详解** | 32位二进制、网络位/主机位划分 |
| **与运算计算** | IP AND 掩码 = 网络地址 |
| **网络地址划分** | 可用IP数量计算、特殊地址 |
| **CIDR写法** | /24、/25、/26等不同表示法 |
| **默认网关** | 连接不同网段的设备（路由器） |
| **DHCP分配** | 动态主机配置协议、租期概念 |
| **DNS协议** | 域名解析系统、域名结构 |
| **Host文件** | 本地域名映射、最高优先级 |

#### 重点概念

- **与运算**：1 AND 1 = 1, 1 AND 0 = 0
- **可用IP计算**：2^(主机位) - 2
- **CIDR**：/24 = 255.255.255.0
- **网关**：不同网段通信的出口
- **DHCP**：自动分配IP，有租期限制
- **DNS**：域名解析服务
- **Host优先级**：本地文件 > 缓存 > DNS服务器

### ③ 上午：ARP协议与文字编码

**📅 日期**: 2026-03-28上午
**📁 文件**: [[05-网络安全课程/③ 上午 - ARP协议与文字编码.md]]

#### 知识点

| 主题 | 说明 |
|-----|------|
| **文字编码基础** | ASCII → GB2312 → GBK → Unicode → UTF-8 |
| **ARP协议原理** | 地址解析协议、广播请求、单播回复 |
| **ARP缓存表** | IP与MAC对应关系、自动更新机制 |
| **ARP欺骗攻击** | 伪造ARP应答、篡改缓存表 |
| **广播风暴** | 成因与广播域划分方法 |

#### 重点概念

- **文字编码演变**：ASCII(1字节) → GB2312(2字节中文) → GBK → Unicode(万国码) → UTF-8(可变长)
- **ARP功能**：将IP地址解析为MAC地址
- **ARP缓存**：每15-45秒更新一次
- **ARP欺骗**：伪造应答将流量导向攻击者
- **广播风暴**：多设备广播导致网络拥堵
- **广播域划分**：通过路由器配合IP地址段划分

### ③ 下午：TCP/UDP与三次握手四次挥手

**📅 日期**: 2026-03-28下午
**📁 文件**: [[05-网络安全课程/③ 下午 - TCPUDP与三次握手四次挥手.md]]

#### 知识点

| 主题 | 说明 |
|-----|------|
| **TCP协议详解** | 面向连接、可靠传输、三次握手 |
| **UDP协议特点** | 无连接、快速传输、不可靠 |
| **三次握手原理** | SYN → SYN-ACK → ACK (双向确认收发能力) |
| **四次挥手原理** | FIN-ACK → ACK → FIN-ACK → ACK (确保数据完整) |
| **TCP/UDP区别** | 可靠性 vs 实时性 |
| **2MSL等待机制** | 保证最后ACK到达 |
| **序列号与确认号** | SEQ和ACK用于有序传输 |

#### 重点概念

- **三次握手**：确认客户端和服务器的收发能力正常
- **四次挥手**：确保数据完整传输后再关闭连接
- **SYN包**：连接请求包(seq=x)
- **ACK包**：确认回应包(ack=x+1)
- **FIN包**：断开连接包
- **2MSL**：等待2倍最大报文生存时间
- **为什么四次挥手**：服务器需要先确认再发送剩余数据
- **为什么不能三次**：为了处理半关闭场景
