## 🗣️ 语言特点（必读）

### 1. 高频表达（去语气词后）

| 原话 | 精简后 | 作用 |
|-----|-------|------|
| "对不对？对不对？" | "对不对？" | 确认理解 |
| "对吧？对吧？" | "对吧？" | 寻求反馈 |
| "然后然后" | "然后" | 衔接 |
| "这个这个" | "这个" | 强调 |

### 2. 自问自答模板

```
老师提问："..."
老师讲解："..."
（学生思考）
老师解答："..."
```

**示例**：
> "子网页码为啥一定是二五？不一定它不一定是 25.25.0。我们后面会讲。"

**要点**：赵老师喜欢先抛出问题，再自己解答，模拟教学对话。

### 3. 口语化表达（保留风格）

| 保留 | 说明 |
|-----|------|
| "happy happy" | 网络课程/直播间 |
| "接着" | "然后"的口语 |
| "先干嘛呢？" | "首先，我们要..." |
| "搞" | "进行" |

## 🎯 教学流程（必背）

### 标准讲解流程

```
1. 提出概念 → "我们先来讲一下XXX"
2. 举例说明 → "比如说这个是一家公司..."
3. 画图演示 → "还是来看一个图吧"
4. 学生练习 → "给大家几分钟做一下"
5. 动手要求 → "动笔动笔动笔写一下"
6. 答案公布 → "OK 公布答案"
7. 详细解析 → "我们来画，我们来写一下"
8. 总结归纳 → "所以说..."
```

### 练习环节流程

```
[开问] "然后给大家弄一个小练习..."
[给时 | "给大家几分钟做一下"
[催促 | "动笔动笔动笔写一下"
[等待 | "5分钟就来讲"
[解答 | "OK 公布答案，它是不可以的"
[分析 | "我们来画，我们来写一下"
[总结 | "所以说他们是不能通信的"
```

## 🖼️ 图解习惯

| 图解类型 | 说明 |
|---------|------|
| **网络拓扑图** | 用"点X"表示通配符（如192.168.1.X） |
| **二进制演示** | "8位二进制连在一起"强调格式 |
| **流量走向** | 箭头表示，强调"不是这个网段，传给路由器" |
| **分层封装** | "一层一层加上去" |

**常用过渡语**：
- "我们画个图吧"
- "拿这个图来看"
- "放大一点"
- "还是来看一个图吧"

## ✅ 整理笔记时的检查清单

每次整理赵老师笔记时，检查是否包含：

- [ ] **图解**：是否保留了画图描述？
- [ ] **例子**：是否保留了具体例子（如192.168.1.6）？
- [ ] **练习**：是否保留了"给大家几分钟做一下"等练习环节？
- [ ] **自问自答**：是否还原了"老师提问→老师解答"的模式？
- [ ] **重复强调**：是否保留了"对不对？对不对？"这类强调？
- [ ] **口语风格**：是否保留了"happy happy"、"接着"等口语？
- [ ] **动画描述**：是否保留了"动笔动笔动笔写一下"等动作描述？
