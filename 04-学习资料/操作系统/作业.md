# Web 服务器集群负载均衡模拟系统说明文档

## 1. 系统概述

本模拟系统通过多线程技术模拟现代 Web 服务器集群的并发处理能力。系统由"模拟服务器"和"负载均衡器"两个核心组件构成，模拟高并发场景下的资源调度与分配。

## 2. 核心架构设计

| 组件 | 类名 | 职责 |
|------|------|------|
| 模拟服务器 | `WebServer` | 继承自线程基类，使用 `time.sleep()` 模拟请求处理耗时 |
| 负载均衡器 | `LoadBalancer` | 充当资源管理器，负责对处理机资源实施有效管理与分配 |

## 3. 调度算法：时间片轮转（Round Robin）

| 特性 | 说明 |
|------|------|
| 核心逻辑 | 将就绪服务器视为循环队列，按顺序指派请求 |
| 公平性保证 | 采用取余运算 `(index + 1) % N` 实现循环指派 |
| 算法优势 | 不要求预知任务时长，更适合 Web 请求场景 |

## 4. 同步机制：互斥锁保护

| 项目 | 说明 |
|------|------|
| 临界区 | 负载均衡器更新"当前服务器索引"的代码段 |
| 保护机制 | 使用 `threading.Lock()` 互斥量 |
| 操作流程 | 分发请求前执行加锁，确保操作原子性 |

## 5. 性能评估准则

| 指标 | 说明 |
|------|------|
| 响应性能 | 确保每个请求在最短时间内得到响应 |
| 系统吞吐量 | 利用多线程并行处理，提升单位时间处理总数 |

## 核心功能说明

### WebServer 类（模拟服务器）
- 继承自 `threading.Thread`，每个服务器作为独立线程运行
- 使用 `queue.Queue` 存储待处理请求
- `time.sleep()` 模拟请求处理耗时
- 提供 `is_busy()` 方法检查服务器状态

### LoadBalancer 类（负载均衡器）
- 实现时间片轮转（Round Robin）算法
- 使用 `threading.Lock()` 保护 `current_index` 更新（临界区）
- 优先分配请求给空闲服务器，保证负载均衡

### Request 类（HTTP请求）
- 包含请求ID、客户端IP、处理时间等属性
- 记录创建时间、开始时间、完成时间用于统计

## 技术要点

| 操作系统概念 | 实现方式 |
|-------------|----------|
| 多线程 | 每个服务器是 `Thread` 子类 |
| 临界区 | 用 `Lock` 保护 `current_index` |
| 调度算法 | 时间片轮转 (Round Robin) |
| 进程同步 | 队列+锁实现安全传递 |

# 核心功能解读

## 系统两大核心设计

### 1. 多线程并发处理架构

系统采用多线程设计，每个 WebServer 实例都是一个独立的线程：

```python
class WebServer(threading.Thread):
    def run(self):
        while self.active:
            request = self.request_queue.get(timeout=0.1)
            self.process_request(request)
```

**OS 知识点对应**：线程是操作系统调度的基本单位，多个线程并发执行，提高了系统的并发处理能力。

### 2. 时间片轮转（Round Robin）调度算法

负载均衡器使用 Round Robin 算法将请求均匀分配给服务器：

```python
self.current_index = (self.current_index + 1) % self.num_servers
```

**OS 知识点对应**：Round Robin 是操作系统中经典的调度算法，通过时间片轮转实现公平调度。

### 3. 互斥锁保护临界区

当多个请求同时竞争 `current_index` 时，使用互斥锁保证数据一致性：

```python
with self.lock:
    self.current_index = (self.current_index + 1) % self.num_servers
```

**OS 知识点对应**：临界区保护是并发编程的基础，通过 Lock 实现互斥访问，防止竞态条件。
