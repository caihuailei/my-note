## 第1章 操作系统引论

### 1.1 操作系统定义

操作系统是**管理和控制计算机硬件与软件资源**的系统软件，是用户与计算机硬件之间的接口。

### 1.2 操作系统的特征

| 特征 | 说明 |
|------|------|
| **并发** | 多个程序同时运行（宏观并行，微观串行） |
| **共享** | 系统资源供多个进程共同使用 |
| **虚拟** | 通过技术手段将一个物理实体变为多个逻辑实体 |
| **异步** | 进程执行速度不可预测，以不可知的速率向前推进 |

### 1.3 操作系统的功能

- **处理器管理**（进程调度）
- **内存管理**（内存分配与回收）
- **设备管理**（I/O设备管理）
- **文件管理**（文件存储、访问、保护）
- **用户接口**（命令接口、程序接口）

## 第2章 进程管理

### 2.1 进程与程序的区别

| 对比项 | 程序 | 进程 |
|--------|------|------|
| 存储方式 | 静态（存放在磁盘） | 动态（程序的一次执行） |
| 生命周期 | 无 | 有（创建→运行→终止） |
| 并发性 | 不能并发 | 可并发执行 |
| 资源拥有 | 不拥有资源 | 拥有资源 |

### 2.2 进程的状态

```
        创建
          ↓
      ┌──┴──┐
      │就绪 │←──────┐
      └──┬──┘       │
         ↓ 时间片用完│
      ┌──┴──┐   ┌──┴──┐
      │运行 │   │阻塞 │
      └──┬──┘   └──┬──┘
         ↓ I/O完成  ↑
      └──┴──┘
       退出
```

**状态说明：**

- **就绪状态**：拥有除CPU外的所有资源，等待调度
- **运行状态**：正在CPU上执行
- **阻塞状态**：等待某事件（如I/O完成）而暂停执行
- **挂起状态**：被移出内存至磁盘，释放物理内存

### 2.3 进程控制块 (PCB)

PCB是操作系统管理进程的核心数据结构，包含：

```
进程描述信息：进程名、进程标识符、用户名
处理机状态信息：通用寄存器、指令计数器、程序状态字、栈指针
进程调度信息：进程状态、优先级、阻塞原因
进程控制信息：程序入口地址、外存地址、资源占有信息、链接指针
```

### 2.4 进程的创建与撤消

**创建原语工作：**
1. 申请空白PCB
2. 为进程分配资源
3. 初始化PCB（描述信息、处理机状态、控制信息）
4. 将新进程插入就绪队列

**撤消原语工作：**
1. 查找进程PCB
2. 终止进程执行并调度
3. 终止所有子孙进程
4. 归还资源
5. 从队列移出

### 2.5 进程的阻塞与唤醒

**引起阻塞的事件：**
- 请求系统服务
- 启动某种操作（如I/O）
- 数据尚未到达
- 无新工作可做

### 2.6 线程

**线程的定义：**
线程是操作系统进行运算调度的最小单位，被包含在进程之中。

**线程与进程的比较：**

| 特性 | 线程 | 进程 |
|------|------|------|
| 资源开销 | 小（共享进程资源） | 大（独立资源） |
| 独立性 | 共享内存，一个出错全部崩溃 | 相互隔离，一个崩溃不影响其他 |
| 调度单位 | CPU调度的基本单位 | 资源分配的基本单位 |
| 通信方式 | 共享内存（需同步） | IPC机制（管道、消息队列等） |

**线程实现方式：**
- **用户级线程**：由线程库实现，切换快但无法利用多核
- **内核级线程**：由内核管理，可并行但切换开销大
- **混合模型**：如Go的GMP调度器

## 第3章 进程同步与通信

### 3.1 同步与互斥

**互斥**：多个进程不能同时使用同一资源

**同步**：多个进程按规定时序执行，共同完成任务

### 3.2 临界资源与临界区

```
进入区 → 临界区 → 退出区 → 剩余区
```

**同步机制准则：**
- **空闲让进**：资源空闲时允许进入
- **忙则等待**：资源正在使用时必须等待
- **有限等待**：保证在有限时间内进入
- **让权等待**：不能进入时释放CPU

### 3.3 信号量与PV操作

**记录型信号量定义：**
```c
struct semaphore {
    int value;           // 资源数量
    struct PCB *queue;   // 等待队列
}
```

**PV操作：**
```
P(s):  s.value = s.value - 1
       若 s.value < 0, 则阻塞

V(s):  s.value = s.value + 1
       若 s.value ≤ 0, 则唤醒
```

### 3.4 经典同步问题

#### 生产者-消费者问题

```c
// 信号量设置
semaphore mutex = 1;   // 互斥访问缓冲区
semaphore empty = n;   // 空缓冲区数量
semaphore full = 0;    // 满缓冲区数量

// 生产者
P(empty);
P(mutex);
// 生产产品放入缓冲区
V(mutex);
V(full);

// 消费者
P(full);
P(mutex);
// 从缓冲区取出产品
V(mutex);
V(empty);
```

#### 读者-写者问题

**读写规则：**
- 多个读者可同时读
- 读时写者不可写
- 写时其他读者和写者均不可访问

#### 哲学家进餐问题

**问题：** 5个哲学家围坐，每人左右各一支筷子，需两支才能进餐

**解决方案（AND信号量）：**
```c
semaphore chopstick[5] = {1,1,1,1,1};

// 使用Swait一次性申请两支筷子
Swait(chopstick[i], chopstick[(i+1)%5]);
eat();
Ssignal(chopstick[i], chopstick[(i+1)%5]);
```

### 3.5 进程通信

**通信类型：**
- **共享存储器系统**：映射共享内存区域
- **消息传递系统**：直接/间接通信
- **管道通信**：共享文件作为通信媒介

**常用方式：**
- 命名管道
- 套接字 (Socket)
- 远程过程调用 (RPC)

## 第4章 调度与死锁

### 4.1 调度类型

| 调度类型 | 别名 | 任务 |
|----------|------|------|
| 高级调度 | 作业调度 | 将外存作业调入内存 |
| 中级调度 | 对换程序 | 内外存之间进程对换 |
| 低级调度 | 进程调度 | 从就绪队列选择进程运行 |

### 4.2 调度算法

#### 先来先服务 (FCFS)

**特点：** 易实现，但不利于短作业

#### 短作业优先 (SJF)

**特点：** 平均周转时间短，但长作业可能饥饿

#### 最短剩余时间优先 (SRTN)

**特点：** SJF的抢占式版本，新任务剩余时间更短则抢占

#### 高响应比优先 (HRRN)

```
响应比 = (等待时间 + 服务时间) / 服务时间
```

**特点：** 避免长作业饥饿

#### 时间片轮转 (RR)

**关键参数：** 时间片大小 q
- q 小 → 响应快，切换开销大
- q 大 → 切换开销小，响应慢

#### 多级反馈队列

**特点：**
- 设置多个就绪队列，优先级逐级降低
- 各队列时间片递增
- 新进程进入第一队列
- 仅当前队列空时调度下一队列

### 4.3 死锁

**定义：** 一组进程因竞争资源而相互等待，若无外力作用将永远不能继续执行

**产生条件（四个必要条件）：**
1. **互斥条件**：资源不能同时共享
2. **请求与保持条件**：已保持资源时又申请新资源
3. **不剥夺条件**：资源不能被强制剥夺
4. **环路条件**：存在进程-资源的环形等待链

### 4.4 死锁的解决方法

#### 死锁预防（预保守策略）

1. **预先静态分配**（破坏请求保持）
   - 进程一次性申请全部资源

2. **剥夺资源**（破坏不剥夺）
   - 当新请求被拒时，释放已占资源

3. **资源有序分配**（破坏环路）
   - 资源编号，按递增顺序申请

#### 死锁避免（动态策略）

**安全状态：** 存在安全序列 `<P1, P2, ..., Pn>`

**银行家算法：**
1. 进行资源预分配
2. 实施安全检测
3. 安全 → 真正分配；不安全 → 回滚

#### 死锁检测与解除

**检测方法：** 资源分配图简化
- 若图可完全简化 → 无死锁
- 若图不可完全简化 → 死锁

**解除方法：**
- 剥夺资源
- 撤消进程（按最小代价原则）

#### 鸵鸟算法

将头埋在沙子里，假装什么也没发生。适用于死锁发生概率很低的场景。

## 💡 重点例题

### Bernstein条件（判断进程能否并发）

```
R(P1) = {a1, a2, ..., am}  // P1读取的变量
W(P1) = {b1, b2, ..., bm}  // P1写入的变量

P1和P2可并发 ⇔
R(P1)∩W(P2) ∪ R(P2)∩W(P1) ∪ W(P1)∩W(P2) = ∅
```

### 示例分析

**例1：** P1: a=5; P2: b=6
- P1、P2可以并发（无交集）

**例2：** P3: c=a+b; P4: d=c+1
- P3、P4不能并发（R(P4)∩W(P3) = {c}）

## 📝 关键公式总结

| 公式 | 说明 |
|------|------|
| 周转时间 = 完成时间 - 到达时间 | 作业执行总时间 |
| 带权周转时间 = 周转时间 / 服务时间 | 相对等待时间 |
| 响应比 = (等待时间 + 服务时间) / 服务时间 | HRRN算法核心 |
