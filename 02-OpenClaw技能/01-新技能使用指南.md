## 1️⃣ agency-agents - 144+ 专家 AI 代理

### 🎯 什么时候使用
当你需要**特定领域的专业 AI 助手**时：
- 前端/后端开发
- UI/UX 设计
- 内容创作
- 营销活动策划
- 销售策略制定
- 产品管理
- DevOps 自动化
- 安全审计
- 社区建设

### 💬 使用方式

#### 激活单个 Agent
```
"使用 Frontend Developer Agent 审查这个 React 组件"
"激活 Content Creator Agent 帮我写博客文章"
"让 Security Engineer 审计这段代码"
"请 Growth Hacker 设计用户增长策略"
```

#### 多 Agent 协作
```
"组建一个团队：Frontend Developer + Backend Architect + UX Designer 来构建新功能"
"使用 Growth Hacker + Content Creator + Twitter Engager 执行营销活动"
```

### 🎭 核心 Agent 推荐

#### 开发相关
- **Frontend Developer** - React/Vue/Angular, UI 实现，性能优化
- **Backend Architect** - API 设计，数据库架构，可扩展性
- **AI Engineer** - ML 模型，部署，AI 集成
- **DevOps Automator** - CI/CD，基础设施自动化
- **Security Engineer** - 威胁建模，安全代码审查

#### 设计相关
- **UI Designer** - 视觉设计，组件库，设计系统
- **UX Researcher** - 用户测试，行为分析，研究
- **Brand Guardian** - 品牌身份，一致性，定位

#### 营销相关
- **Growth Hacker** - 快速获取用户，病毒循环，实验
- **Content Creator** - 多平台内容，编辑日历
- **SEO Specialist** - 技术 SEO，内容策略，外链建设
- **Reddit Community Builder** - 真实互动，价值驱动内容

#### 销售相关
- **Outbound Strategist** - 基于信号的 prospecting，多渠道序列
- **Deal Strategist** - MEDDPICC 资格认证，竞争定位
- **Sales Engineer** - 技术演示，POC 范围界定

## 2️⃣ AutoResearchClaw - 全自动学术研究

### 🎯 什么时候使用
当你需要**完整的学术研究**时：
- 从想法到论文的自动化研究
- 文献综述和综合
- 实验设计和执行
- 论文撰写和格式化
- 引用验证
- 多代理同行评审

### 💬 使用方式

#### 基本用法
```
"Research quantum computing error correction codes"
"Research the impact of learning rate schedules on transformer convergence"
"Research diffusion models for ICML 2026 submission"
```

#### 高级用法
```
"Research transformer attention mechanisms with MetaClaw enabled"
"Research the future of AI safety with full experiment execution"
```

### 🔄 23 阶段流程（自动执行）

1. **Topic Init** - 定义研究范围
2. **Problem Decompose** - 分解为研究问题
3. **Search Strategy** - 规划文献搜索
4. **Literature Collect** - 从 OpenAlex、Semantic Scholar、arXiv 收集
5. **Literature Screen** - 质量门禁（人类批准）
6. **Knowledge Extract** - 提取知识卡片
7. **Synthesis** - 聚类发现，识别差距
8. **Hypothesis Gen** - 生成可测试假设（多代理辩论）
9. **Experiment Design** - 设计实验（质量门禁）
10. **Code Generation** - 生成硬件感知代码
11. **Resource Planning** - 估计资源
12. **Experiment Run** - 在沙箱中执行
13. **Iterative Refine** - 自我修复（最多 10 轮）
14. **Result Analysis** - 多代理分析
15. **Research Decision** - PROCEED/REFINE/PIVOT 决策
16. **Paper Outline** - 构建论文结构
17. **Paper Draft** - 逐节撰写
18. **Peer Review** - 多代理评审（证据检查）
19. **Paper Revision** - 根据反馈修订
20. **Quality Gate** - 最终质量检查（人类批准）
21. **Knowledge Archive** - 归档到知识库
22. **Export Publish** - 导出 LaTeX
23. **Citation Verify** - 最终引用验证

### 📦 输出文件

| 文件 | 描述 |
|------|------|
| `paper_draft.md` | 完整学术论文（5000-6500 字） |
| `paper.tex` | 会议就绪的 LaTeX（NeurIPS/ICML/ICLR 模板） |
| `references.bib` | 真实 BibTeX 引用 |
| `verification_report.json` | 4 层引用验证报告 |
| `experiment runs/` | 生成的代码 + 结果 |
| `charts/` | 自动生成的条件比较图表 |
| `reviews.md` | 多代理同行评审 |
| `evolution/` | 自学习经验 |
| `deliverables/` | 所有最终输出 |

### ⚙️ 要求
- LLM API 密钥（OpenAI、Anthropic 等）
- 可选：Semantic Scholar API 密钥
- 可选：Docker 用于沙箱执行
- 可选：Obsidian Publish 用于知识库

## 3️⃣ CLI-Anything - 为软件生成 CLI

### 🎯 什么时候使用
当你需要**控制桌面应用程序**时：
- 为任何软件创建 CLI
- 自动化创意工具（GIMP、Blender、OBS）
- 管理办公软件（LibreOffice）
- 自动化视频/音频编辑
- 控制通信工具（Zoom）

### 💬 使用方式

#### 构建 CLI
```
"Use CLI-Anything to build a CLI for GIMP"
"Create a CLI harness for ./blender"
"Generate CLI for LibreOffice"
"Build a CLI for this GitHub repo: https://github.com/.../..."
```

#### 细化 CLI
```
"Refine the GIMP CLI to add more batch processing commands"
"Expand the Blender CLI with more 3D modeling capabilities"
```

#### 验证 CLI
```
"Validate the LibreOffice CLI against HARNESS.md standards"
"Test the OBS Studio CLI"
```

### 🏗️ 7 阶段流水线

1. **Analyze** - 扫描源代码，映射 GUI 操作到 API
2. **Design** - 架构命令组、状态模型、输出格式
3. **Implement** - 构建 Click CLI，支持 REPL、JSON 输出、撤销/重做
4. **Plan Tests** - 创建 TEST.md，包含单元+E2E 测试计划
5. **Write Tests** - 实现全面的测试套件
6. **Document** - 更新 TEST.md 与测试结果
7. **Publish** - 创建 `setup.py`，安装到 PATH

## ⚠️ 注意事项

### agency-agents
- 每个 Agent 都是独立的专家，明确指定你需要的 Agent
- 对于复杂任务，可以激活多个 Agent 协作
- 参考原始项目获取更多 Agent 详情

### AutoResearchClaw
- 需要 LLM API 密钥
- 研究过程可能需要较长时间（取决于课题复杂度）
- 确保有稳定的网络连接以获取文献
- 建议先从小型课题开始测试

### CLI-Anything
- 目标软件需要已安装在系统中
- 生成 CLI 后需要 `pip install` 才能使用
- 某些软件可能需要额外的依赖
- 建议先测试生成的 CLI 是否正常工作
