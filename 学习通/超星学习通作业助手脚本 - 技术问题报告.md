# 超星学习通作业助手脚本 - 技术问题报告

---

## 一、项目概述

**目标**：开发一个书签脚本，实现超星学习通平台的自动答题功能。

**功能范围**：
- 选择题（单选/多选）
- 判断题
- 填空题

**技术栈**：纯 JavaScript（书签脚本），运行于浏览器环境

---

## 二、遇到的问题及解决方案

### 2.1 多选题选项识别问题

| 问题编号 | PROBLEM-001 |
|---------|------------|
| **问题描述** | 第 54 题正确答案为 ABCDEF，但脚本只能识别 ABCD，EF 选项丢失 |
| **根本原因** | 正则表达式和验证逻辑只支持 `[A-D]` 范围 |
| **影响范围** | 所有 5-6 个选项的多选题 |
| **解决方案** | 将所有 `[A-D]` 正则改为 `[A-F]`：<br>• `getLbl` 函数中的选项标签匹配<br>• `parse` 函数中的答案解析正则<br>• `extractCorrect` 函数中的答案提取正则 |
| **验证方式** | 测试 6 选项多选题，确认 EF 能被正确识别和填写 |

---

### 2.2 填空题编辑器无法写入

| 问题编号 | PROBLEM-002 |
|---------|------------|
| **问题描述** | 填空题答案无法写入输入框，测试和正式填写均失败 |
| **根本原因** | 学习通使用 UEditor 富文本编辑器，内容存储在 iframe 中，存在跨域限制，不能直接操作 DOM |
| **尝试方案** | <table><tr><td>方案</td><td>结果</td><td>原因</td></tr><tr><td>操作 textarea.value</td><td>❌</td><td>textarea 是隐藏的，内容不同步</td></tr><tr><td>操作 iframe.contentWindow.document</td><td>❌</td><td>跨域限制，无法访问</td></tr><tr><td>UE.instants[taId]</td><td>❌</td><td>key 不是 textarea ID</td></tr><tr><td>UE.instants[ueditorInstantX]</td><td>✅</td><td>需要用索引映射</td></tr></table> |
| **最终方案** | 通过 `UE.instants` 索引访问编辑器实例：<br>`const keys = Object.keys(window.UE.instants);`<br>`const editor = window.UE.instants[keys[idx]];`<br>`editor.setContent(answer);` |
| **关键发现** | `ueditorInstant0` → `answerEditor4055241291`（索引对应，非名称对应） |

### 2.3 面板无法滚动

| 问题编号 | PROBLEM-003 |
|---------|------------|
| **问题描述** | 脚本面板内容超出屏幕时，无法查看下方按钮 |
| **根本原因** | 未设置最大高度和滚动属性 |
| **解决方案** | 添加 CSS 样式：<br>`max-height: 60vh;`<br>`overflow-y: auto;` |

### 2.4 AI 答案格式解析失败

| 问题编号 | PROBLEM-004 |
|---------|------------|
| **问题描述** | AI 返回的答案格式多样，脚本无法正确解析 |
| **具体表现** | <ul><li>`- 选择题：1.C` → 无法识别</li><li>`第 1 空：ALTER COLUMN 第 2 空：ORDER BY` → 只能识别一个</li><li>`1.A` → 正常识别</li></ul> |
| **解决方案** | 扩展正则表达式支持多种格式：<br>`/[-•]?\s*(?:选择题 | 多选题 | 判断题)?\s*[:：]?\s*(\d+)\s*[\.,.:\u3001\s:：]+\s*([A-FABab]+)/`<br><br>支持格式：<br>• `1.A`<br>• `- 选择题：1.C`<br>• `- 多选题：18.ABCD`<br>• `- 判断题：26.A` |

### 2.5 填空题答案格式不匹配

| 问题编号 | PROBLEM-005 |
|---------|------------|
| **问题描述** | 用户粘贴的答案格式与脚本期望格式不一致 |
| **用户格式** | `第 1 空：ALTER COLUMN 第 2 空：ORDER BY`（一行内多个） |
| **脚本期望** | 每行一个答案，按顺序 |
| **解决方案** | 修改 `parseBlank` 函数，支持三种格式：<br>1. 一行内多个：`第 1 空：XXX 第 2 空：XXX`<br>2. 每行带题号：`第 1 空：XXX`<br>3. 纯答案列表：`XXX`（按顺序匹配） |

---

### 2.6 剪贴板权限被拒绝

| 问题编号 | PROBLEM-006 |
|---------|------------|
| **问题描述** | `navigator.clipboard.writeText()` 执行失败，报错 "Write permission denied" |
| **根本原因** | 浏览器安全策略限制，HTTPS 页面或某些设置阻止脚本自动访问剪贴板 |
| **解决方案** | 改为弹窗显示文本，用户手动复制：<br>1. 显示题目到只读 textarea<br>2. 提供"全选"按钮<br>3. 提示用户按 `Ctrl+C` 复制 |
| **用户体验** | 增加一步操作，但保证功能可用性 |

---

### 2.7 选项文本提取不完整

| 问题编号 | PROBLEM-007 |
|---------|------------|
| **问题描述** | 第 7 题等部分题目选项显示为空，AI 无法作答 |
| **根本原因** | 学习通页面结构中，选项文本可能不在第一个 `<p>` 标签中：<br>• 正常：`<p>选项内容</p>`<br>• 异常：`<p></p><p>选项内容</p>`（第一个 p 为空） |
| **原代码** | `const te = o.querySelector('.answer_p p');`<br>只获取第一个 `<p>` 标签 |
| **解决方案** | 新增 `getOptText` 函数，遍历所有 `<p>` 和 `<div>` 标签，合并所有非空文本：<br>`const ps = o.querySelectorAll('.answer_p p,.answer_p div,p,div');`<br>`for(let i=0;i<ps.length;i++){ if(ps[i].innerText.trim()) text+=ps[i].innerText.trim()+' '; }` |
| **验证方式** | 检查第 7 题选项是否正确显示在 AI 提示词中 |

## 三、技术架构总结

### 3.1 核心函数

| 函数名 | 作用 | 关键实现 |
|--------|------|----------|
| `getQCs()` | 获取所有题目 | `querySelectorAll('.padBom50.questionLi.fontLabel.singleQuesId')` |
| `getOpts(c)` | 获取选项元素 | `querySelectorAll('[onclick*="addChoice"],.answerBg,[role="radio"]')` |
| `getLbl(o)` | 获取选项标签 (A-F) | 读取 `data` 属性或 `aria-label` |
| `getOptText(o)` | 获取选项文本 | 遍历所有 `<p>` 和 `<div>` 合并文本 |
| `getBlanks()` | 获取填空题信息 | 遍历 `.stem_answer .Answer` 提取 textarea |
| `fillBlank(idx,ans)` | 填写填空题 | `UE.instants[keys[idx]].setContent(ans)` |
| `parseAllAnswers(txt)` | 解析 AI 答案 | 正则匹配选择题和填空题 |

## 四、经验教训

### 4.1 第三方编辑器处理
- **教训**：不要尝试直接操作 iframe 或隐藏 textarea
- **正确做法**：查找并使用官方 API（如 `UE.instants[key].setContent()`）

### 4.2 跨域问题
- **教训**：iframe 跨域时无法访问 `contentWindow.document`
- **正确做法**：通过父页面暴露的 API 间接操作

### 4.3 实例映射关系
- **教训**：不能假设实例 key 的命名规则
- **正确做法**：通过调试打印确认映射关系，使用索引匹配

### 4.4 DOM 结构变化
- **教训**：页面结构可能因题目类型、版本更新而变化
- **正确做法**：使用多种选择器兜底，遍历所有可能容器

### 4.5 浏览器权限
- **教训**：剪贴板 API 在某些环境下不可用
- **正确做法**：提供降级方案（手动复制）

### 4.6 用户输入格式
- **教训**：用户粘贴的格式可能与预期不同
- **正确做法**：支持多种格式，正则表达式要足够灵活
