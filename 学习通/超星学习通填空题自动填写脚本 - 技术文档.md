# 超星学习通填空题自动填写脚本 - 技术文档

---

## 一、需求说明

**目标**：创建一个书签脚本，实现以下功能：
1. 自动识别超星学习通页面的所有填空题
2. 复制题目给 AI 获取答案
3. 将 AI 返回的答案自动填写到对应的填空框中

**支持的答案格式**：
- `第 1 空：ALTER COLUMN 第 2 空：ORDER BY`（一行内多个）
- `第 1 空：ALTER COLUMN`（每行一个带题号）
- `ALTER COLUMN`（纯答案，每行一个，按顺序）

---

## 二、页面结构分析

### HTML 结构
```html
<div class="questionLi fontLabel singleQuesId">
  <h3 class="mark_name">23. <span>(填空题)</span> 题目内容...</h3>
  <div class="stem_answer">
    <div class="Answer">
      <span class="tiankong">第 1 空</span>
      <div class="edui-default">
        <textarea id="answerEditor4055241291"></textarea>
        <iframe id="ueditor_0"></iframe>
      </div>
    </div>
  </div>
</div>
```

### 编辑器结构
- 使用 **UEditor** 富文本编辑器
- 每个填空对应一个 `textarea`（隐藏）和一个 `iframe`（显示）
- 编辑器实例存储在 `window.UE.instants` 中
- 实例 key 为 `ueditorInstant0`、`ueditorInstant1`...（按页面顺序）

## 三、遇到的问题及解决方案

### 问题 1：无法直接操作 iframe

| 尝试方法 | 代码 | 结果 | 原因 |
|---------|------|------|------|
| 操作 textarea | `textarea.value = '答案'` | ❌ | textarea 是隐藏的，内容不同步 |
| 操作 iframe DOM | `iframe.contentWindow.document.body.innerHTML` | ❌ | 跨域限制 |
| 使用 UE API | `UE.instants[key].setContent()` | ✅ | 正确方式 |

### 问题 2：编辑器实例 key 映射错误

**错误代码**：
```javascript
const key = taId.replace('answerEditor', 'ueditorInstant');
// 生成：ueditorInstant4055241291 ❌
```

**实际情况**：
```
ueditorInstant0 -> answerEditor4055241291
ueditorInstant1 -> answerEditor4055241301
ueditorInstant2 -> answerEditor4055241302
```

**正确代码**：
```javascript
const keys = Object.keys(window.UE.instants);
const editor = window.UE.instants[keys[idx]];  // 按索引匹配 ✅
```

### 问题 3：答案格式解析

**支持的格式及解析逻辑**：
```javascript
// 格式 1：第 X 空：答案
const regex = /第\s*(\d+)\s*空\s*[：:]\s*([^第]+)/g;
// 提取：{1: 'ALTER COLUMN', 2: 'ORDER BY'}

// 格式 2：每行一个
const lines = txt.split(/\n|;/).map(x => x.trim()).filter(x => x);
```

### 问题 4：面板无法滚动

**解决**：
```css
max-height: 60vh;
overflow-y: auto;
```

## 六、核心函数说明

| 函数 | 作用 | 关键代码 |
|------|------|----------|
| `getBlanks()` | 获取所有填空信息 | 遍历 `.questionLi`，提取题号、空号、textarea ID |
| `fill(idx, ans)` | 填写答案 | `UE.instants[keys[idx]].setContent(ans)` |
| `parseAnswers(txt)` | 解析答案文本 | 正则匹配 `第 X 空：答案` 或按行分割 |

## 八、技术总结

**核心教训**：
1. 第三方富文本编辑器（UEditor、TinyMCE 等）必须使用其官方 API 操作
2. 不要尝试直接操作 iframe 或隐藏的 textarea
3. 实例映射关系需要通过调试确认，不能假设命名规则

**成功关键**：
- 正确识别 `UE.instants` 的 key 与填空的索引对应关系
- 支持多种答案格式，适应用户不同的粘贴习惯
- 提供测试按钮，快速验证填写功能是否正常
